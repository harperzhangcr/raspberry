# 飞书创新委看板自动生成链路

## 目标

将飞书表单/多维表格中的创新课题数据，经过 JSON 抽取与 Python 规则化处理，稳定生成创新管理委员会 HTML 看板。

## 全链路

1. 飞书表单/多维表格取数
   - 源数据包括课题清单、里程碑、本月目标、当前状态、关键成果、交付物、下月计划、风险、PMO 评价、展示材料、指摘落实等。

2. 飞书智能助手抽取 JSON
   - 飞书侧负责把原始表单内容抽取为 JSON。
   - Python 不直接访问飞书 API，而是读取飞书智能助手产出的 JSON 文件。

3. 运行 `Fetch_Data.py`
   - 自动计算报表周期：当前日期减 15 天，得到目标月份、季度、月末截止日和会议期次。
   - 读取目标月份 JSON，例如 `进度跟踪看板-202605存档_data.json`。
   - 读取课题清单、课题风险与指摘落实、历次创新委指示等 JSON。
   - 生成 12 个 CSV 中间结果：
     - `里程碑完成率指标统计表.csv`
     - `指摘落实.csv`
     - 10 个课题各自的本月进度表 CSV
   - 在内存中完成字段清洗、统计、排序、状态图标、风险颜色、PMO 文案、展示材料链接等规则化处理。
   - 读取 HTML 模板，并只替换动态数据区域。
   - 输出最终 HTML 看板到 `artifacts` 目录。

4. 参考 HTML 模板生成看板
   - HTML 模板负责样式、布局、字体和页面结构。
   - `Fetch_Data.py` 负责数据注入、模块增减、统计计算和排序。

## 主要输入

- `进度跟踪看板-YYYYMM存档_data.json`
- `课题清单.json`
- `课题风险与指摘落实（课题组）.json`
- `历次创新委课题&部门指示.json`
- `第7次创新管理委员会.html`

## 主要输出

默认输出到 `artifacts`：

- 12 个 CSV 中间结果
- 最终 HTML 看板，例如 `第7次创新管理委员会.html`

## 运行方式

在工程目录下运行：

```bash
python3 Fetch_Data.py
```

如果需要指定 HTML 模板路径：

```bash
DASHBOARD_TEMPLATE_PATH="/path/to/template.html" python3 Fetch_Data.py
```

如果需要指定输出 HTML 路径：

```bash
DASHBOARD_OUTPUT_PATH="/path/to/output.html" python3 Fetch_Data.py
```

## 当前关键规则

- 第 1、2 页日期、季度、会议期次自动根据当前日期减 15 天计算。
- 第 2 页 KPI、课题风险、完成率来自里程碑完成率统计。
- 第 3 页指摘落实表按同一相关课题/部门分组排序：
  - 事项类型：先 `指摘`，后 `建议`
  - 指摘提出人：`丁国祥` 优先，其次 `李化`，再其他人
  - 建议提出人：`何修华` 优先，再其他人
- 第 4-13 页按 10 个课题数据自动生成：
  - 本月里程碑模块支持按数据条数自动增减
  - 关键成果与交付物模块支持按数据条数自动增减
  - 下月行动计划模块支持按数据条数自动增减
  - 风险模块支持按数据条数自动增减；为空时显示 `暂无本月风险项`
  - PMO 总结为空或风险等级正常时显示 `正常推进`
  - 展示材料字段存在 URL 时，在下月行动计划前插入成果展示模块

## 设计原则

AI 只负责飞书侧 JSON 抽取；看板生成阶段由 Python 做确定性处理，避免每次让 AI 直接读表导致的不稳定。
