# 对话工作记录：飞书创新委看板自动化

## 背景

原先看板主要依赖 AI 直接读表生成，每次稳定性不够。目标是把飞书侧抽取出的 JSON 交给 Python 做确定性处理，再把算好的数据熔炼进 HTML 模板，形成稳定的创新委汇报看板。

## 主要讨论与决策

1. 数据链路确认
   - 用户已有飞书智能助手负责从飞书表单/多维表格抽取 JSON。
   - Python 不负责飞书 API 取数，只读取本地 JSON。
   - `Fetch_Data.py` 负责字段清洗、统计、排序、规则化和 HTML 生成。

2. CSV 与内存变量关系
   - 讨论后确认：CSV 可以作为中间结果和校验产物，但最终 HTML 不需要再靠浏览器读取 CSV。
   - `Fetch_Data.py` 跑完后，可以直接用 Python 内存里的结构化数据生成 HTML。

3. 第 1、2 页动态化
   - 根据当前日期减 15 天计算报表月份、季度、月末截止日和会议期次。
   - KPI、完成率、提前完成数、延期数、课题风险、进度条等改为由里程碑统计表生成。

4. 第 3 页指摘落实
   - 指摘落实表改为从 JSON/CSV 的指摘落实模块生成。
   - 后续新增排序规则：
     - 同一相关课题/部门内，先 `指摘` 后 `建议`
     - 指摘提出人：`丁国祥` 优先，其次 `李化`
     - 建议提出人：`何修华` 优先

5. 第 4-13 页课题详情
   - 参照第 4 页模板，批量生成 10 个课题详情页。
   - 本月里程碑、关键成果与交付物、下月行动计划、风险与 PMO 总结均支持根据原始数据条数自动增减。

6. 风险与 PMO 规则
   - 风险为空时显示 `暂无本月风险项`。
   - PMO 总结和风险等级为空或正常推进时，显示 `正常推进`。
   - 非正常推进时保留风险等级与 PMO 评价组合显示。

7. 展示材料字段
   - 在课题清单 JSON 中新增读取 `[展示材料]` URL。
   - 如果 URL 存在，在下月行动计划前插入“成果展示”模块。

8. 模板与输出路径
   - 考虑飞书环境运行，HTML 模板默认与 `Fetch_Data.py` 同目录。
   - 输出 HTML 默认与 12 个 CSV 一起放入 `artifacts`。
   - 支持用环境变量指定模板路径和输出路径。

9. 样式小调整
   - 第 2 页进度统计文字数字加粗，并按已完成、提前、延期做颜色区分。
   - 封面会议标题、图标、公司名、截止日期字号放大。
   - 曾尝试在第 4-13 页里程碑与交付物外加透明卡片，后按用户要求回退。
